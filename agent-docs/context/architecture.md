## Native module architecture

All AEP packages expose a React Native native module. As of 2026, the `optimize` package is being migrated to **TurboModules** (New Architecture). All others still use the old bridge.

### Old bridge pattern (most packages)

```
JS: NativeModules.AEP<Module>.<method>()
    ↓
iOS:  RCTAEPModule : RCTEventEmitter  (RCT_EXPORT_MODULE)
Android: RCTAEPModule extends ReactContextBaseJavaModule
```

### TurboModule pattern (optimize, in progress)

```
JS spec: specs/NativeAEP<Module>.ts   (Codegen source of truth)
    ↓ Codegen generates native interfaces
JS: TurboModuleRegistry.getEnforcing('NativeAEP<Module>')
    ↓
iOS:  RCTAEP<Module>.mm : NSObject <NativeAEP<Module>Spec>
         getTurboModule: → NativeAEP<Module>SpecJSI
Android: NativeAEP<Module>Module extends NativeAEP<Module>Spec
```

### Dual-mode toggle: USE_INTEROP_ROOT

Allows a single package to serve both New Arch (Turbo) and old bridge (Interop) consumers without two separate builds.

| Value | iOS | Android |
|-------|-----|---------|
| `0` / `false` | Turbo: `NSObject <NativeAEP<Module>Spec>` | Turbo: `NativeAEP<Module>Module` |
| `1` / `true` | Interop: `RCTEventEmitter <NativeAEP<Module>Spec>` + `RCT_EXPORT_MODULE` | Interop: legacy bridge module |

- **iOS**: compile-time (`GCC_PREPROCESSOR_DEFINITIONS` via podspec env var)
- **Android**: runtime (`BuildConfig.USE_INTEROP_ROOT` in `build.gradle`)

See `migrations/optimize-turbo.md` for the full pattern.

## iOS build specifics

- `use_frameworks! :linkage => :static` — required for AEP SDK (Swift frameworks)
- `OTHER_SWIFT_FLAGS = '-no-verify-emitted-module-interface'` — applied to all AEP targets in Podfile `post_install`
- `RCTAppDependencyProvider.{h,mm}` — generated by RN codegen during `pod install` into `apps/AwesomeProject/ios/build/generated/ios/ReactAppDependencyProvider/`; never delete `ios/build` after pod install
