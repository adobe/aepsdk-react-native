## iOS — Quick commands

| What | Command |
|------|---------|
| Build + run (Turbo Module) | `export NVM_DIR="$HOME/.nvm" && source "$NVM_DIR/nvm.sh" && yarn e2e:ios:build:turbo` |
| Build + run (Interop Layer) | `export NVM_DIR="$HOME/.nvm" && source "$NVM_DIR/nvm.sh" && yarn e2e:ios:build:interop` |
| Run only (no rebuild) | `export NVM_DIR="$HOME/.nvm" && source "$NVM_DIR/nvm.sh" && yarn e2e:ios` |

> **Always** source nvm before running — wdio spawns appium as a subprocess and needs it on PATH.

## USE_INTEROP_ROOT: switching modes

Turbo (`USE_INTEROP_ROOT=0`) and Interop (`USE_INTEROP_ROOT=1`) paths produce different iOS build artifacts. **You must clean when switching.**

The `e2e:ios:build:turbo` / `e2e:ios:build:interop` scripts do this automatically:
1. `rm -rf Pods Podfile.lock build` (inside `apps/AwesomeProject/ios/`)
2. `USE_INTEROP_ROOT=<n> pod install`
3. `xcodebuild`
4. `wdio`

> **Do not** add a second `rm -rf ios/build` after pod install — codegen writes `RCTAppDependencyProvider.h` into `ios/build/` during pod install (pre-install hook). Deleting it after will break xcodebuild. See `errors/e2e-ios-build-rctappdependencyprovider-not-found.md`.

---

## Codegen files (RN 0.84+)

- `RCTAppDependencyProvider.{h,mm}` is generated by RN codegen during `pod install`
- Output: `apps/AwesomeProject/ios/build/generated/ios/ReactAppDependencyProvider/`
- If you ever delete `ios/build` manually after pod install, re-run `pod install` before building

## All yarn e2e scripts (reference)

| Script | What it does |
|--------|-------------|
| `yarn e2e` | Run wdio only (no build). Auto-detects platform. |
| `yarn e2e:build` | Build app + run wdio. Auto-detects platform. |
| `yarn e2e:ios` | wdio iOS only |
| `yarn e2e:ios:build` | Build (default mode) + wdio iOS |
| `yarn e2e:ios:build:turbo` | Clean → pod install (USE_INTEROP_ROOT=0) → xcodebuild → wdio iOS |
| `yarn e2e:ios:build:interop` | Clean → pod install (USE_INTEROP_ROOT=1) → xcodebuild → wdio iOS |
| `yarn e2e:android:build` | `assembleDebug` + wdio Android |
